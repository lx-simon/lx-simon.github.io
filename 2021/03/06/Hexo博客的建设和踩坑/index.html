Hexo博客的建设和踩坑

创建新的博客

hexo new “新博客名”
hexo new draft “新草稿”

在_posts 同级目录 _drafts产生草稿

如图：

博客文章内容

格式

写文章的格式和CSDN类似，照着格式写即可。
自己多写几篇就熟悉了
文件里面按markdown的标准写
CSDN写文章和hexo写文章

写文章

在你创建博客的根目录
在/blog/source/_posts找到博客名.md,直接vi它，即可进入写博客的界面，按照格式写博客即可。

文章内插入图片

  • 图片插入格式

  • 第一步，安装插件，在博客根目录下执行

    npm install –save hexo-asset-image npm install hexo-asset-image –save

  • 打开根目录博客的配置文件_config.yml

    点击Esc,输入/post_asset_folder回车，将false改为true

  • 此后创建新的博客在_posts目录下就会有同名的文件夹。

** 如果还是不行的话 **

  • 在博客根目录下 /node_modules/hexo-asset-image/index.js
    将里面所有内容替换下面的代码 如果你不放心上面代码，你可以先备份一下

    mv index.js bindex.js
    然后自己创建一个index.js把代码写入

图片插入格式和说明

以后创建新博客，hexo new ‘name’ 会在/source/_posts 下创同名的name文件夹，之前的博客也想插入可mkdir 手动创建
格式

拓展名不一定是jpg ,看自己

博客文章图片设置

我的themes是butterfly！!
下面会写怎么配置主题。
博客开头top_img: ~~图片路径或者~~url地址,cover: ~~图片路径或者~~url地址。

~~图片路径为：/博客名/图片名 ~~

图片要放在同名的文件夹内

top_img是博客文章内顶部图片，cover是博客主页看到的图片。

themse主题

  • 这里选用的是butterfly主题
  • 在博客的根目录下

    git clone -b master https://github.com/jerryc127/hexo-theme-butterfly.git themes/butterfly
    如果上面执行不成功，可以换成：
    git clone -b master https://gitee.com/iamjerryw/hexo-theme-butterfly.git themes/butterfly,注意分支这里是master

  • 也可以不执行上面的，直接执行

    npm i hexo-theme-butterfly

  • npm安装此只支持Hexo 5.0.0以上版本*
  • 修改博客根目录下的配置_config.yml
  • vi后，保持Esc状态，/theme,回车即可直接找到theme*

    theme: butterfly

  • 如果你沒有pug以及stylus的渲染器，請下載安裝：

如果报错模块不存在！！

  • 将node_module文件夹删掉，重新安装
    如图(这里看有*号的命令我是删掉lib库里的node_module)：
  • 将博客根下node_module文件夹删掉重新安装配置。
  • *可以看图中命令参考参考**，备份后一切重来也无妨，详见我的第一篇博客。

butterfly主题默认图片更改。

  • 默认的cover和top_img
  • 更改默认：

    在博客根目录/themes/butterfly/scripts/filters/random_cover.js
    vi 进入random_cover.js,找到cover更改为自己喜欢的即可。

不过还是自己些博客的时候，在顶部配top_img和cover更好。

  • 图片路径/img

    /指的是在该目录source目录下

主页图片的更改

  • 在themes下改配置文件，找到Image，下面的index.img,改成自己喜欢的url或者本地路径(/img/图片名称)

  • 配置文件有注释的，直接看注释就能知道是什么文件。像index.img，首页图片，注释为the banner image of home page
