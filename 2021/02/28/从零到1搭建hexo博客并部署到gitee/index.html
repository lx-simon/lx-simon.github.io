从零到1搭建hexo博客并部署到gitee

发表于2021-02-28|更新于2021-05-02
|阅读量:

使用Termux搭建博客并发布

我的博客：我的博客

地址：https://lx-simon.gitee.io

前言

搭建博客的教程网上特别的多，这篇基本上没太多自己的想法。
我写这博客是想熟悉一下操作

背景

这里是用hexo的静态博客框架，然后用termux，一个高级终端。
可以在手机上部署和上传博客，其实在电脑和手机上都大同小异
的，我是将博客部署到gitee。

环境的准备

  • Termux可以在官网上直接下载，地址：https://termux.com/,下载F-Droid安装即可。
    不过注意官方已经停止更新play版了，大家可以在酷安找找，也能下载。不建议直接在下
    酷安的，可以去评论翻翻。

  • 更新一下源

    apt update && apt-get update

  • 安装git

    apt-get install git -y

  • nodejs 和 npm

    apt-get install nodejs npm -y

查看版本，判断安装是否成功

node -v

npm -v

  • 安装Hexo

mkdir blog && cd blog && npm install -g hexo-cli

npm install -g hexo-cli安装hexo，前面的创建文件夹，放配置文件的。

初始化hexo

cd ~ && hexo init blog && cd blog && npm install

cd ~ && hexo init blogblog在 ~ 目录中，blog要为空，初始化的操作。会在blog添加一些配置文件
cd blog && npm install打开blog，安装依赖包。

启动本地预览

hexo g && hexo s

hexo g会生成文件到本地public文件夹
hexo s启动本地服务器，并未发出，是自己看一下自己写的。
执行成功会显示http://localhost:4000，在自己部署的手机的浏览器查看。

Ctrl+C停止本地预览

termux的路径为/data/data/com.termux/files/home,使用ls能看到blog文件夹，cd blog
进入blog文件夹内。termux一打开就在home目录下，pwd可以显示当前你a在哪个目录。

要是想玩termux可以去看看国光大佬的教程
国光大佬的termux教程：https://www.sqlsec.com/2018/05/termux.html

对了，blog里的_config.yml就是hexo的配置文件，想改就直接vi _config.yml。报错就直接百度，很多教程的，感兴趣的
也可以自己百度找一些，慢慢完善自己的博客。

gitee的设置

先登录gitee官网<gitee.com>,先登录，没账号的注册。
gitee都是中文，自己创建一个和自己用户名相同的账号仓库,否则需要改配置。

如图：

如果你创了一个新仓库，仓库是空的，下面会有一段代码，直接复制在终端中执行就好了(初始化git）

生成ssh并添加到gitee

ssh-keygen -t rsa -C”youremail”

cat ~/.ssh/id_rsa.pub

注意：生成ssh时第一次回车就行，输入值的话会改变ssh密钥的名称。

cat 到的全部复制，然后点左上角 gitee的图标 ，然后点用户名旁边的 个人主页 ，再从左边 找到ssh公钥

粘贴确定即可。

回到termux
ssh -T git@gitee.com查看是否连接。

修改配置

部署

点击上图 服务 的位置，会有gitee pages, 直接点击启动就可以。

注意，在本地生成、部署后，都需要点gitee pages里的更新才能够同步blog，可能会慢几分钟，不急。

安装deploy-git

在termux执行：

npm install –save hexo-deployer-git npm install hexo-deployer-git –save

然后再清除缓存，生成、部署一下：

hexo clean && hexo g -d

最后打开gitee pages中的网址，应该和本地预览的一样的。

如果首页格式乱了没图片，应该是仓库没同名，网上查一下怎么配置就好了。
